Module 5 · BI Layer

Direct Lake Semantic Model + Power BI

Build a Direct Lake semantic model on the Bronze Lakehouse from Module 2 and spin up a 1-page Power BI report on top. No import, no DirectQuery overhead - Power BI reads your Delta files in OneLake directly, with live data freshness.

⏱ 30 minutes
🎯 Goal: live 1-page report
🛠 Direct Lake · Power BI

Learning Objectives

  • Create a Direct Lake semantic model on the Bronze Lakehouse.
  • Define relationships between your warm-path tables.
  • Author three core DAX measures.
  • Auto-create and polish a 1-page Power BI report.
  • Verify the model is actually running in Direct Lake mode (not DirectQuery fallback).

Direct Lake Semantic Model + Power BI DIRECT LAKE

ⓘ
What is Direct Lake? A Power BI storage mode that reads Delta files in OneLake directly - no import, no DirectQuery overhead. You get import-mode performance with live data freshness, and the semantic model is just metadata pointing at your Lakehouse.

  1. Create the semantic model

    From lh_urbanpulse_bronze, click the New semantic model button at the top of the Tables view.

    In the picker, tick:

    • Students
    • Classes
    • Enrollments
    • cosmos_grades
    • tumor_metadata

    Name it: sm_urbanpulse_education. Click Confirm.

    ✓
    For the Smart City scenario we're using education tables as a stand-in for any "warm path" data - the same Direct Lake pattern applies whether your tables are admissions, sales, or sensors.

  2. Define relationships

    Open the semantic model → Model view. Drag-and-drop to create:

    Relationships 
    Enrollments[StudentId]   ──*──1── Students[StudentId]
Enrollments[ClassId]     ──*──1── Classes[ClassId]
cosmos_grades[Student_ID] ──*──1── Students[StudentId]

  3. Add three measures

    In the model, right-click Enrollments → New measure. Repeat for each:

    DAX 
    Total Enrollments = COUNTROWS(Enrollments)

Avg Grade =
    AVERAGEX(
        cosmos_grades,
        VALUE(cosmos_grades[Grade])
    )

Pass Rate % =
    DIVIDE(
        CALCULATE(COUNTROWS(cosmos_grades), VALUE(cosmos_grades[Grade]) >= 60),
        COUNTROWS(cosmos_grades),
        0
    ) * 100

  4. Auto-create the report

    Top of the semantic model → Create report → Auto-create. Power BI builds a starter page using your measures and dimensions.

  5. Polish the report

    Edit the auto-created page so it has three visuals:

    1. A card with Total Enrollments
    2. A bar chart: Avg Grade by ClassName
    3. A donut: count of students by class

    Save as rpt_urbanpulse_education.

    📷

  6. Validate Direct Lake mode

    In the semantic model settings, the Storage mode field for each table should read Direct Lake. If it's "DirectQuery" instead, your capacity might not support Direct Lake - flag a coach.

Validation

  • Semantic model sm_urbanpulse_education exists in your workspace.
  • Three relationships are drawn in Model view.
  • All three DAX measures evaluate without errors.
  • Report rpt_urbanpulse_education has at least three visuals.
  • All tables report Direct Lake storage mode.

What You Just Built

  • A Direct Lake semantic model over your Bronze Lakehouse - no import refresh, no DirectQuery latency.
  • Relationships and three DAX measures defined.
  • A 1-page Power BI report on live OneLake data.
💡
Why this matters for ISVs. Direct Lake lets you ship BI on top of your customer's Fabric data without designing a separate analytic database. Power BI is the analytic engine, and OneLake is the storage. Less infrastructure, faster time to insight.
