Setup & Environment Check

Everything in this lab happens in your browser - the Microsoft Fabric portal and the Azure portal. No Cloud PC, no VS Code, no Python installs. Let's confirm your pre-provisioned workspace and grab the Lab Credentials page you'll reference all day.

⏱ 10 minutes
🎯 Goal: green lights everywhere
🛠 Browser only

Learning Objectives

  • Sign in to the Microsoft Fabric portal with your lab account.
  • Verify your pre-provisioned workspace and capacity assignment.
  • Bookmark the Lab Credentials reference page (Appendix A).
  • Optional: confirm the four shared Azure Event Hubs are healthy in the Azure portal.

Your Pre-Provisioned Environment

Each attendee receives an isolated stack on top of a shared, coach-managed data plane.

Asset What & Why Identifier
Fabric Workspace Your private workspace where you'll build all artifacts today. urbanpulse-{your-id}
Capacity (F-SKU) Shared trial capacity across all attendees in the room. fc-isvsummit-{room}
Lakehouse (empty) Bronze landing zone - you'll populate this in Module 2. lh_urbanpulse_bronze
Eventhouse (empty) Real-Time Intelligence destination - you'll create KQL tables in Module 3. eh_urbanpulse_rti
Shared Event Hubs (4) Coaches run the simulators centrally on a hidden VM. You'll connect Eventstreams to these in Module 3 using read-only listener strings. see Appendix A
Lab Credentials Page One reference page (Appendix A) with every connection string, key, and endpoint you'll need to paste into Fabric portal wizards today. Appendix A
Azure OpenAI + AI Search Used by the optional Foundry orchestrator in Module 8. aoai-isvsummit + srch-cityops
ⓘ
Why pre-provisioned? Day 2 is about building Fabric solutions, not waiting on capacity assignments, quota approvals, or device registration. Coaches provisioned everything overnight so you can hit the ground running.

Step 1 - Sign in to Microsoft Fabric

  1. Open the Fabric portal in your browser

    On your own laptop (or any device), open app.fabric.microsoft.com.

    Sign in with the lab account credentials on the printed slip at your seat:

    Lab Account 
    username:  attendee-{id}@isvsummitlab.onmicrosoft.com
password:  see your slip
    ✓
    Use a private / InPrivate window if you're already signed into your work tenant - it avoids identity confusion all day long.

  2. Switch to your workspace

    In the left navigation, click Workspaces → search for urbanpulse- and click the one matching your attendee ID.

Validation

You should see a workspace named urbanpulse-{your-id} containing two empty items: lh_urbanpulse_bronze (Lakehouse) and eh_urbanpulse_rti (Eventhouse). The capacity badge in the top-right should read F-SKU (not "Trial").

Step 2 - Open the Lab Credentials reference page

Every connection string, key, and endpoint you'll need today lives on a single page in this guide. Bookmark it now so you can paste from it into Fabric portal wizards as you go.

  1. Open Appendix A in a new tab

    Click here: Appendix A · Lab Credentials

    It contains the values for:

    • The four shared Event Hub listener strings (Module 3)
    • Azure SQL connection details for the mirror (Module 2)
    • Cosmos DB endpoint + read-only key for the OneLake Shortcut (Module 2)
    • Azure OpenAI endpoint + key (Module 8 Foundry orchestrator - optional)
    • Azure AI Search endpoint + index name (Module 8 - optional)

  2. Bookmark it in your browser

    Press Ctrl+D on the Lab Credentials tab and save it to a folder named ISV Summit Lab. You'll come back to this page in every module from M2 onward.

ⓘ
Why a reference page instead of a config file? You're building everything in the Fabric portal - no local code, no .env files, no installs. A single page with all the values means one place to look, one place to update.

Step 3 (optional) - Confirm the streams are live

If you want to see the simulators running before Module 3, open the Azure portal and navigate to the Event Hubs namespace listed in Appendix A. Each of the four hubs should show non-zero Incoming Messages in the last 5 minutes.

✓
Skip this step if you're behind on time. Module 3 will validate streams in the Fabric Eventstream UI anyway.
