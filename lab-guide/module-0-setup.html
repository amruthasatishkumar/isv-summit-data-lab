Setup & Environment Check

Every step in this lab is performed in a browser - the Microsoft Fabric portal and the Azure portal. No Cloud PC, VS Code, or local Python install is required. This module summarizes the environment that is provisioned for you in advance and the items you build across Modules 1-9.

⏱ 10 minutes
🎯 Goal: green lights everywhere
🛠 Browser only

Learning Objectives

  • Understand what's pre-provisioned versus what you'll build across Modules 1-9.

What's Pre-Provisioned vs. What You'll Build

The lab account, the Fabric capacity, the Fabric workspace, and the Azure data plane are pre-provisioned. You populate the workspace - lakehouse, eventhouse, dashboards, agents - across Modules 2-9.

Asset What & Why Identifier
Fabric Capacity (F-SKU) (provided) Shared Fabric capacity allocated to this lab. Already attached to your workspace. fc-isvsummit-{room}
Lab Account (provided) Lab tenant identity that has Contributor on the capacity above. Fabric.UserN@estdcorp.onmicrosoft.com
Fabric Workspace (provided) One Fabric workspace per lab user, already bound to the shared capacity. You'll see it under Workspaces as soon as you sign in to Fabric. per-user (visible in Fabric)
Shared Event Hubs (3) (provided) Three streaming sources run continuously in the lab environment. You'll connect Eventstreams to them in Module 3 using a listen-only SAS connection string issued by your coach. see Lab Credentials
Azure AI Foundry project (provided, optional use) Each lab account has its own Foundry project (with gpt-4o-mini already deployed) for the optional Module 10 orchestrator extension. visible at ai.azure.com
Lab Credentials Page (provided) One reference page listing each lab user account and where to find the per-user Azure resources (SQL, Cosmos, Foundry) in the Azure portal. Lab Credentials
Items inside the workspace (you build) Lakehouse + Mirror + Shortcut (M2), Eventhouse + Eventstreams + KQL tables (M3), Real-Time Dashboard (M4), Direct Lake semantic model + Power BI report (M5), Fabric Ontology (M6), Fabric Data Agent (M7). Optional: extra agents (M8) and Foundry orchestrator (M9). Use any names you like - the guide suggests one convention but it isn't enforced. your naming, your ID
ⓘ
Why this provisioning split? You learn Fabric by building in Fabric. The shell - capacity, identity, workspace, and shared streams - is pre-provisioned so you can spend the day on the parts that matter: lakehouse, eventhouse, dashboards, semantic model, ontology, and agents. Pick whatever names you like for the items you build; the guide suggests one convention, but feel free to deviate.
