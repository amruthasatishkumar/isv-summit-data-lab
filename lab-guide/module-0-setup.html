Module 0 · Foundation

Setup & Environment Check

Everything in this lab happens in your browser - the Microsoft Fabric portal and the Azure portal. No Cloud PC, no VS Code, no Python installs. Let's confirm your lab capacity assignment and grab the Lab Credentials page you'll reference all day. You'll create your workspace and every Fabric item from scratch as you go.

⏱ 10 minutes
🎯 Goal: green lights everywhere
🛠 Browser only

Learning Objectives

  • Sign in to the Microsoft Fabric portal with your lab account.
  • Verify the lab Fabric capacity is assigned to your account (you'll bind your workspace to it in Module 1).
  • Bookmark the Lab Credentials reference page (Appendix A).
  • Optional: confirm the four shared Azure Event Hubs are healthy in the Azure portal.

What's Pre-Provisioned vs. What You'll Build

Coaches set up the bare minimum: a shared Fabric capacity, an Azure data plane, and a credentials sheet. You build the workspace and every Fabric item.

Asset What & Why Identifier
Fabric Capacity (F-SKU) (provided) Shared trial capacity across all attendees in the room. You'll bind your workspace to this in Module 1. fc-isvsummit-{room}
Lab Account (provided) Lab tenant identity that has Contributor on the capacity above. attendee-{id}@isvsummitlab.onmicrosoft.com
Shared Event Hubs (3) (provided) Coaches run the simulators centrally on a hidden VM. You'll connect Eventstreams to these in Module 3 using read-only listener strings. see Appendix A
Azure OpenAI + AI Search (provided, optional use) Used by the optional Foundry orchestrator in Module 9. aoai-isvsummit + srch-cityops
Lab Credentials Page (provided) One reference page (Appendix A) with every connection string, key, and endpoint you'll paste into Fabric portal wizards today. Appendix A
Workspace + items (you build) Workspace (M1), Lakehouse + Mirror + Shortcut (M2), Eventhouse + Eventstreams + KQL tables (M3), Real-Time Dashboard (M4), Direct Lake semantic model + Power BI report (M5), Fabric Ontology (M6), Fabric Data Agent (M7). Optional: extra agents (M8) and Foundry orchestrator (M9). your naming, your ID
ⓘ
Why capacity-only provisioning? You learn Fabric by building in Fabric. Coaches handle the things that need a tenant admin (capacity, identity, shared streams). Everything else - the workspace itself, the lakehouse, the eventhouse, the dashboards, the agents - is yours to create. That's the muscle memory you take home.

Step 1 - Sign in to Microsoft Fabric

  1. Open the Fabric portal in your browser

    On your own laptop (or any device), open app.fabric.microsoft.com.

    Sign in with the lab account credentials on the printed slip at your seat:

    Lab Account 
    username:  attendee-{id}@isvsummitlab.onmicrosoft.com
password:  see your slip
    ✓
    Use a private / InPrivate window if you're already signed into your work tenant - it avoids identity confusion all day long.

  2. Confirm the lab capacity is assigned to you

    You don't have a workspace yet - you'll build one in Module 1. For now, just confirm Fabric loaded with no errors and that you can see the Power BI / Fabric home.

    If you want a deeper check, click your profile picture (top-right) → Account manager. The capacity fc-isvsummit-{room} should appear in the list of capacities you have access to.

Validation

You're signed in to Fabric as attendee-{id}@isvsummitlab.onmicrosoft.com, the home page loads with no errors, and you have access to the lab capacity. Workspaces and items are intentionally empty right now - you'll create them in the modules to come.

Step 2 - Open the Lab Credentials reference page

Every connection string, key, and endpoint you'll need today lives on a single page in this guide. Bookmark it now so you can paste from it into Fabric portal wizards as you go.

  1. Open Appendix A in a new tab

    Click here: Appendix A · Lab Credentials

    It contains the values for:

    • The three shared Event Hub listener strings (Module 3)
    • Azure SQL connection details for the mirror (Module 2)
    • Cosmos DB endpoint + read-only key for the OneLake Shortcut (Module 2)
    • Azure OpenAI endpoint + key (Module 9 Foundry orchestrator - optional)
    • Azure AI Search endpoint + index name (Module 9 - optional)

  2. Bookmark it in your browser

    Press Ctrl+D on the Lab Credentials tab and save it to a folder named ISV Summit Lab. You'll come back to this page in every module from M2 onward.

ⓘ
Why a reference page instead of a config file? You're building everything in the Fabric portal - no local code, no .env files, no installs. A single page with all the values means one place to look, one place to update.

Step 3 (optional) - Confirm the streams are live

If you want to see the simulators running before Module 3, open the Azure portal and navigate to the Event Hubs namespace listed in Appendix A. Each of the three hubs should show non-zero Incoming Messages in the last 5 minutes.

✓
Skip this step if you're behind on time. Module 3 will validate streams in the Fabric Eventstream UI anyway.
