Azure Resources (per-user)

Each lab account owns its own pre-provisioned Azure resources. Sign in to portal.azure.com with your Fabric.UserN@estdcorp.onmicrosoft.com credentials, open the resource group assigned to your user, and pick up the connection strings, hostnames, and access keys directly from each resource's blade.

ResourceWhere to find it in the portalUsed in
Azure SQL Database
Hospitals, Wards, Staff		 Resource group → SQL database resource → Connection strings blade for the server name and database name; Settings → SQL admin login for the credentials. Public network access and Allow Azure services are pre-configured. Module 2 · Mirror
Azure Cosmos DB
regions, trainRoutes		 Resource group → Cosmos DB account → Settings → Keys for endpoint and primary key. The urbanpulse database and the regions + trainRoutes containers are pre-created. Module 2 · Mirror
Azure Event Hubs
medicalvitals, medicalmovement, metrotrain		 Producers run centrally for the room; you only consume from these hubs. Namespace, hub names, and a listen-only SAS connection string are issued by your coach at the start of Module 3. Module 3 · Eventstreams
Azure AI Foundry project
(Module 10, optional)		 Sign in to ai.azure.com with the same lab account. Your pre-provisioned project appears on the home screen. The gpt-4o-mini deployment is already attached - no keys to copy. Module 10 · Foundry (optional)
