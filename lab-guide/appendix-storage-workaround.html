Storage Account Workaround

Use this path if you cannot create a OneLake shortcut to the shared storage account for the Module 2 facility-catalog exercise. Instead of mounting the files from Azure Blob Storage, download them and upload them into your own Lakehouse.

ⓘ
When to use this. The default Module 2 path creates a OneLake shortcut to the shared safabricshareduse2dev storage account (Option A). If you do not have access to that storage account or the SAS token has expired, use the download-and-upload route below to reach the same end state.

Step 1 · Get the files

The facility catalog Parquet file and the five facility photos are bundled with this lab guide.

  1. Open the Download & Upload Data appendix.
  2. Click Download all (.zip) to grab everything in one archive, or download the individual files.
  3. Extract the archive. You should end up with facility_catalog.parquet next to a facilities/ folder containing five JPGs.

Step 2 · Upload into your Lakehouse

  1. In the Fabric portal open lh_urbanpulse_bronze (or whatever name you gave your Bronze Lakehouse in Module 2).
  2. In the Explorer pane right-click Files → Upload → Upload files and pick facility_catalog.parquet.
  3. Right-click Files again → Upload → Upload folder and pick the facilities folder containing the five JPGs.

Step 3 · Load the catalog to a table

  1. Right-click facility_catalog.parquet → Load to tables → New table.
  2. Accept the auto-detected schema and name the table facility_catalog (or whatever you prefer).
  3. Continue with the rest of Module 2 using this uploaded data in place of the storage-account shortcut.
✓
After this you'll have one Delta table (facility_catalog) and one folder of binary assets (Files/facilities/) - the same end state as the OneLake shortcut, just copied into your Lakehouse instead of mounted.

Validation

  • facility_catalog.parquet and the facilities/ folder appear under Files in your Lakehouse.
  • The facility_catalog table exists under Tables with 5 rows.
  • The photo_path values resolve to files under Files/facilities/.
