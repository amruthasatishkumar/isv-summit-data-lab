UrbanPulse:
Build a Smart City Operations Platform on Microsoft Fabric

A guided, scenario-based hands-on lab where you assume the role of an ISV building a real-time, multi-domain operations platform - combining live hospital, transit, and airspace data into a unified Fabric solution with natural-language data agents.

Duration
~6 hours
Modules
7 (M0 – M6)
Skill level
Intermediate
Environment
Pre-provisioned
MODULE 0

Setup & Environment Check

Verify your pre-provisioned Fabric workspace, capacity, lakehouse, eventhouse, and the four shared Azure Event Hubs are wired up. Confirm everything is ready before we start building.

15 min · UI tour, sanity checks
 MODULE 1

Workspace Tour & ISV Architecture

Walk through the UrbanPulse multi-tenant architecture, OneLake explorer, capacity, RBAC, and the medallion model you'll build today.

30 min · concepts + UI tour
 MODULE 2

Lakehouse, Shortcuts & Mirroring

Land tumor-imaging Parquet into the Bronze Lakehouse, mirror an Azure SQL Student DB with zero-ETL, and shortcut a Cosmos DB container for unified data access.

45 min · Lakehouse + Mirror + Shortcut
 MODULE 3

Real-Time Streaming Ingest

Stand up four Eventstreams routing live data from Event Hubs into a Fabric Eventhouse - hospital vitals, hospital movement, Metra trains, and live ADS-B aircraft.

75 min · Event Hubs → Eventstream → Eventhouse
 MODULE 4

KQL Analytics & Real-Time Dashboard

Write KQL across hospital, transit, and airspace tables. Pin live tiles to a Real-Time Intelligence Dashboard and watch the city update in front of you.

60 min · KQL + RTI Dashboard
 MODULE 5

Direct Lake + Power BI

Build a Direct Lake semantic model on the Bronze Lakehouse and ship a 1-page Power BI report - no import refresh, no DirectQuery latency.

30 min · Direct Lake + Power BI
 MODULE 6

Build & Publish a Fabric Data Agent

Build the Hospital Operations Fabric Data Agent on live KQL data, then publish it to Microsoft 365 Copilot. Optional stretch: Transit + Airspace agents.

45 min · Fabric Data Agents · M365 Copilot
 MODULE 7 · ★ OPTIONAL

Foundry Connected-Agent Orchestrator

Wire your Fabric Data Agents into an Azure AI Foundry connected-agent orchestrator and publish that to M365 Copilot for cross-domain reasoning. No code.

30 min · Azure AI Foundry · M365 Copilot

Prerequisites

ⓘ

Everything you need has been pre-provisioned. Each attendee receives:

  • A dedicated Fabric workspace (urbanpulse-{your-id}) on a shared F-SKU capacity
  • Read access to four shared Azure Event Hubs streaming live data
  • A Lab Credentials reference page with every connection string, key, and endpoint pre-filled
  • An Azure AI Search index city-ops-index + Azure OpenAI deployment for the optional Module 7 Foundry orchestrator

You do not need your own Azure subscription, a Cloud PC, or any local installs - everything happens in your browser. Coaches run the data simulators centrally; your job is to build on top.
